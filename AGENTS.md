# AGENTS.md

이 문서는 이 저장소에서 작업하는 Codex/에이전트의 기본 작업 규칙을 정의합니다.

## 1) 안전 및 윤리 규칙 (최우선)

- 실제 무기 제어, 유도, 타격, 교전, 살상 목적의 로직/코드 작성 금지.
- 실제 군사 장비 사양, 민감한 프로토콜, 실전 운용 절차를 모사하지 말 것.
- 교육용 mock sensor, mock packet, mock firmware만 허용.
- 모든 예제는 가상 데이터/시뮬레이션 기반으로 작성.

## 2) 저장소 목적 준수

- 목적: 취업 준비생의 코드 독해 역량 강화.
- 단순 장난감 수준 예제 지양, 실무 맥락을 반영한 구조 선호.
- 코드 길이는 학습자가 읽고 분석하기 충분한 규모로 구성.

## 3) C 프로젝트 작성 가이드

- header/source 분리 구조를 기본으로 사용.
- Makefile 포함, `gcc`로 PC 환경 빌드 가능한 형태 권장.
- 사용 권장 개념: pointer, struct, enum, union, bit operation, ring buffer,
  static/extern, volatile(필요 이유 주석), function pointer.
- 방어적 코딩(입력 검증, 경계 체크, 오류 코드 처리) 적용.

## 4) Python 프로젝트 작성 가이드

- class 기반 구조와 type hint 사용.
- dataclass, enum, logging, argparse, pathlib 활용 권장.
- deque/queue 기반 버퍼링/메시지 처리 포함 권장.
- 예외 처리(exception handling) 및 간단한 pytest 단위테스트 포함 권장.

## 5) 문서화 원칙

- 각 프로젝트 README에 아래 항목 포함:
  - 프로젝트 목적
  - 실행 방법
  - 주요 학습 포인트
  - 확장 과제
- 학습자 스스로 분석할 수 있도록 질문 리스트 제공 가능.
- 상세 해설은 요청 시 제공.

## 6) 금지 사항

- 불필요한 복잡성 추가 금지(학습 목적과 무관한 과도한 추상화 지양).
- 보안상 민감하거나 오해 소지가 있는 군사용 세부 구현 금지.

## 7) 권장 품질 기준

- 읽기 쉬운 파일 분할과 명확한 네이밍.
- 로깅/오류 처리/설정 분리 등 실무형 코드 패턴 유지.
- 실행 가능한 mock/simulation 상태를 기본 완료 기준으로 삼기.
